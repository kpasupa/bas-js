Grouped by dialect (BASIC core vs GW-BASIC extensions), then by the official keyword type. Coverage reflects src/interp/basic.js.

Legend
✓ DONE fully working   ◐ PART parsed but simplified / stub   ○ TODO not done, but possible — on the dev plan
⊘ DUMMY safe to fake or no-op (e.g. FRE — modern RAM dwarfs that era)   ⌫ SKIP won't implement — hardware / machine-language, or editor-only (out of scope for a file runner)

BASIC = the portable core common to the Microsoft BASIC family (flow, I/O, math/string functions, operators). GW-BASIC = the IBM-PC / MS-DOS additions (screen, graphics, sound, record & sequential files, event/error trapping, type-conversion).
The remaining ○ TODO items are the ones needing the real browser device layer — graphics (canvas), sound (Web Audio), event traps (DOM keys), SCREEN modes, FILES listing. They need visual/integration checks, so they're deferred past this pass.
Notes: OPEN supports RANDOM + sequential (INPUT/OUTPUT/APPEND). RSET aliases LSET (no right-justify). WIDTH/KEY parse but do nothing. Verified by test/harness.html (85 cases, headless Chrome) — every ✓ above has a test.
